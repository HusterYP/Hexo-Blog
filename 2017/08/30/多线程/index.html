前言

Java多线程

进程

  • 就是应用程序在内存中分配的空间(正在运行中的程序)

线程

  • 是进程中负责执行的执行单元，也叫执行路径；一个进程中至少有一个线程在负责该进程的运行

    多线程

  • 解决多部分代码同时执行的需求，合理的使用CPU资源
  • 多线程的运行时CPU的切换完成的，具有一个随机性

-

  • start方法
> `start`方法可以开启线程并执行`run`方法，注意是两个功能(开启线程，执行`run`)
  • 调用start方法和调用run方法有什么区别

    调用start方法还可以开启一个线程，但是只是调用run方法的话，还是主线程执行的

  • 调用线程的this.getName()和Thread.currentThread().getName()有什么区别

    this指代的是当前对象，获取的当前对象的名字(默认Thread-n),但是Thread.currentThread()获取的是当前执行路径

    如果想上所述子线程没有开启的话，其获取的就是main主线程名字

Runnable接口体现的面向对象思想

  • 根据文档可知实现多线程除了继承Thread类，还可以实现Runnable接口，复写run方法，同时通过Thread的构造函数：Thread(Runnable target)
  • 可知只需要将实现了Runnable的类对象传入即可实现多线程
  • 下面以多窗口售票为例

-

class SaleTicket implements Runnable
{
    private int tickets = 100;
    @Override
    public void run()
    {
        while(true)
        {
            if (tickets>0)
                System.out.println(Thread.currentThread().getName()+"..."+tickets--);
        }
    }
}

public class Demo
{
    public static void main(String[] args)
    {
        SaleTicket saleTicket = new SaleTicket();
        Thread t1 = new Thread(saleTicket);
        Thread t2 = new Thread(saleTicket);
        Thread t3 = new Thread(saleTicket);
        Thread t4 = new Thread(saleTicket);
        t1.start();
        t2.start();
        t3.start();
        t4.start();
    }
}
  • Runnable接口的出现使得多线程各自的任务与线程对象分离，只要符合该规则就可以用多线程，这种思想将任务单独抽离成为类(将线程单独进行对象的封装)，同时实现Runnable接口也使得可以传入同一个对象，使得更符合生活场景(降低了线程对象和线程任务的耦合性)
  • 由于Java中不支持多继承，所以实现Runnable接口可以避免继承Thread类的单继承的局限性

多线程的安全问题

  • 如上的代码示例，修改如下

-

class SaleTicket implements Runnable
{
    private int tickets = 100;
    @Override
    public void run()
    {
        while(true)
        {
            if (tickets>0)
            {
                try //这里为了使问题出现，加了一个休眠，使结果会输出0，-1，-2...
                {
                    sleep(10);
                } catch (InterruptedException e)
                {
                    e.printStackTrace();
                }
                System.out.println(Thread.currentThread().getName() + "..." + tickets--);
            }
        }
    }
}

public class Demo
{
    public static void main(String[] args)
    {
        SaleTicket saleTicket = new SaleTicket();
        Thread t1 = new Thread(saleTicket);
        Thread t2 = new Thread(saleTicket);
        Thread t3 = new Thread(saleTicket);
        Thread t4 = new Thread(saleTicket);
        t4.start();
        t2.start();
        t1.start();
        t3.start();
    }
}
  • 出现类似上面这种安全问题的原因是
    1. 线程任务中有多条处理到共享数据
    2. 线程任务中有多条对共享数据的操作，一个线程在操作共享数据的过程中，其他线程参与了运算，造成了数据的错误

解决上面安全问题的方法

  • 解决思想

    只要保证多条操作共享数据的代码在某一时间段，被一条线程所执行，在执行期间不允许其他线程参与运算

  • 使用synchronize代码块

    同步锁保证一次只能有一个线程在执行

    弊端：

    减低了效率(需要判断锁)，但是在可接受的范围之内

-

class SaleTicket implements Runnable
{
    private int tickets = 100;
    Object obj = new Object();  //这里的obj必须在run方法之外，如果在run中new，那么相当于每一个线程有不同的obj，没有效果(但是obj的作用是什么？)(见下3,4)
    @Override
    public void run()
    {
        while(true)
        {
            synchronized (obj)  //在这里加一个同步锁

    多个线程在同步中必须使用同一个锁，这才是对多线程同步(如上所述obj的new位置)

  • 如果同步函数是静态的呢

    static方法随着类加载，这是不一定有该类的对象，但是一定有一个该类的字节码文件对象(在堆区中,Java也将字节码文件封装成了一个对象，在JavaAPI中可用Class类描述)，可以使用’类名.class’得到该对象，如果同步函数被static修饰，那么默认的就是该字节码对象做同步锁

单例模式中线程问题的

  • 懒汉式(延迟加载模式)，在多线程并发访问时会出现线程安全问题，加了同步就可以解决问题，但是降低效率(可以通过if对单例对象双重判断的形式)；饿汉式相对线程安全

-

------------饿汉式--------------
class Single
{
    private static final Single SINGLE = new Single(); //饿汉式
    private Single()
    {}
    public static Single getInstance()
    {
        return SINGLE;
    }

}
class Fun implements Runnable
{

    public void run()
    {
        Single single = Single.getInstance(); //线程安全(前提只是在获取对象上，如果做了共享数据运算，仍然不安全)
    }
}
--------------------------------
-------------懒汉式-------------
class Single
{
    private static Single s = null;
    private Single()
    {
        if (s == null) //这里通过双重判断，可以减少对同步锁的不断判断
        {
            synchronized (Single.class)//使用字节码对象做锁
            {
                if (s == null) //这里必须还要判断一次，否则仍然会有安全问题
                    s = new Single();
            }
        }
    }
    public static Single getInstance()
    {
        return s;
    }
}
class Fun implements Runnable
{

    public void run()
    {
        Single single = Single.getInstance();
    }
}

同步函数和同步代码块的区别

  • 同步函数只能用this作为锁，同步代码块可以使用任意对象作为锁
  • 如果说一个类中只需要一个锁，这时可以考虑使用同步函数吗，使用this，写法简单;但是如果一个类中需要多个锁，还有多个类中使用同一个锁，这时只能使用同步代码块

  • 死锁的实例

-

class Task implements Runnable
{
    private boolean flag;

    public Task(boolean flag)
    {
        this.flag = flag;
    }

    public void run()
    {
        if (flag)
        {
            while (true)
                synchronized (Lock.LOCKA)
                {
                    System.out.println("if...LOCKA");
                    synchronized (Lock.LOCKB)
                    {
                        System.out.println("if...LOCKB");
                    }
                }
        } else
        {
            while (true)
                synchronized (Lock.LOCKB)
                {
                    System.out.println("else...LOCKA");
                    synchronized (Lock.LOCKA)
                    {
                        System.out.println("else...LOCKB");
                    }
                }
        }
    }
}

class Lock
{
    public static final Object LOCKA = new Object();
    public static final Object LOCKB = new Object();
}

public class Demo
{
    public static void main(String[] args)
    {
        Task t1 = new Task(true);
        Task t2 = new Task(false);
        new Thread(t1).start();
        new Thread(t2).start();
    }
}

多线程间的通信

  • 多个线程都在处理同一个资源，但是处理的任务却不一样
  • 生产者，消费者(生产一个消费一个)(等待/唤醒机制)(允许空唤醒，即notify的时候没有wait线程)

    wait()：让线程处于冻结状态，存储到线程池中

    notify():任意唤醒一个线程(wait()和notify()方法必须要用在同步当中，因为他们是用来操作同步锁上的线程的)(而且wait和notify在使用时，必须标识他们所属于的锁，标识方法是：锁对象.wait(),锁对象.notify(),锁对象那个.notifyAll()

    相同锁的notify(),可以获取相同锁的wait(),也就是说不同的锁不可以唤醒其他锁的wait()线程，相当于每个锁都有一个线程池(这也可以理解为什么wait和notify方法在Object类中了))

-

class Goods
{
    private String name = "Goods";
    private int count = 1;
    private boolean flag = true;  
    //同时要注意的是，wait()和notify()也属于生产和消费的内容

    public synchronized void get()  //这里要用同步，不然会报错
    {
        if(!flag)
            try
            {
                wait();  //等下次被唤醒后，仍从wait这里开始运行，不需要再判断flag,除非进行到下一次while
            } catch (InterruptedException e)
            {
                e.printStackTrace();
            }
        System.out.println("Consumer......" + name + "..." + count);
        flag = false;
        notify();
    }

    public synchronized void set()
    {
        if(flag)
            try
            {
                wait();
            } catch (InterruptedException e)
            {
                e.printStackTrace();
            }
        count++;
        System.out.println("Producer...." + name + "..." + count);
        flag = true;
        notify();
    }
}

class Producer implements Runnable
{
    private Goods goods;

    public Producer(Goods goods)
    {
        this.goods = goods;
    }

    public void run()
    {
        while(true)
            goods.set();
    }
}

class Consumer implements Runnable
{
    private Goods goods;

    Consumer(Goods goods)
    {
        this.goods = goods;
    }

    public void run()
    {
        while(true)
            goods.get();
    }
}

public class Demo
{
    public static void main(String[] args)
    {
        Goods goods = new Goods();
        Producer producer = new Producer(goods);
        Consumer consumer = new Consumer(goods);
        Thread t1 = new Thread(producer);
        Thread t2 = new Thread(consumer);
        t1.start();
        t2.start();
    }
}

解决多生产多消费的效率问题

  • 使用JDK1.5 java.util.concurrent.locks包中的对象

  • Locks接口：他的出现比synchronized有更多的操作

    lock():获取锁

    unlock():释放锁

  • 同步代码块或同步函数的锁操作是隐式的，JDK1.5 Lock接口，按照面向对象的思想，将锁单独封装成了一个对象，并提供了对锁的显示操作

  • Lock接口就是同步的替代

    1. 将程序中的同步更换为Lock接口的形式(看到接口的形式，首先应该找其是否已实现的子类，如果没有再自己复写)

    2. 替换完后运行失败了，因为wait没有了同步区域，没有了所属的同步锁，同步升级了，其中锁已经不是任意对象，而是Lock类型的对象。

    查看文档，Condition接口替代了Object中的监视器方法(以前将监视器封装到每一个对象当中，现在将监视器方法封装到了Condition对象中)

    1. 同过Lock接口的newCondition方法创建一个Condition对象来绑定锁(将监视器方法单独封装成了对象，再与锁相绑定)

-

class Goods
{
    private String name = "Goods";
    private int count = 1;
    private Lock lock = new ReentrantLock();
    private Condition producer_con = lock.newCondition();
    private Condition consumer_con = lock.newCondition();
    private boolean flag = true;  //这里之所以还要一个标志，是为了在开始的时候不知道是哪个线程得到执行权

    public void get()
    {
        lock.lock(); //获取锁
        try
        {
            while (!flag)
                try
                {
                    consumer_con.await();
                } catch (InterruptedException e)
                {
                    e.printStackTrace();
                }
            System.out.println("Consumer......" + name + "..." + count);
            flag = false;
            producer_con.signal(); //在这里唤醒生产者的一个线程，可以避免唤醒全部，提高效率
        } finally
        {
            lock.unlock();  //释放锁，锁一定要释放，所以写在finally中，这里也是try..finally的应用
            //这是为了防止中间产生异常导致代码退出，使锁为释放，造成死锁
        }
    }

    public void set()
    {
        lock.lock();
        try
        {
            while (flag)
                try
                {
                    producer_con.await();
                } catch (InterruptedException e)
                {
                    e.printStackTrace();
                }
            count++;
            System.out.println("Producer...." + name + "..." + count);
            flag = true;
            consumer_con.signal();
        } finally
        {
            lock.unlock();
        }
    }
}

wait()和sleep()的

  • 相同点：都可以让线程处于冻结状态
  • 不同点

    1. wait()可以指定时间，也可以不指定时间；sleep()必须指定时间

    2. wait()：释放CUP资源，同时释放锁；sleep()释放CPU资源，但是不释放锁

如何停止线程

  • 原理：让run方法结束(线程任务中一般都有循环)，只要控制住循环就可以结束run方法，停止线程，用一个标记
  • 中断状态即冻结状态，interrupted()可以强制唤醒处于中断状态的线程，但是会抛出异常，使用该方法常用于通过标志位结束线程时却陷入死锁，此时使用该方法来强制唤醒

-

class Test implements Runnable
{
    private boolean flag = true;

    public synchronized void run()
    {
        while (flag)
        {
            try
            {
                wait(); //在这里陷入死锁
            } catch (InterruptedException e)
            {
                System.out.println(Thread.currentThread().getName() + "..." + e.toString());
                flag = false; //在这里改变标志位也可以
            }
            System.out.println(Thread.currentThread().getName() + "...." + "Hello World");
        }
    }

    public void changeFlag()
    {
        flag = false;
    }
}

public class Demo
{
    public static void main(String[] args)
    {
        Test t = new Test();
        Thread t1 = new Thread(t);
        Thread t2 = new Thread(t);
        t1.start();
        t2.start();
        for (int i = 0; i < 50; i++)
        {
            System.out.println(Thread.currentThread().getName() + "..." + "Hello World");
            if (i == 40)
            {
                t1.interrupt();
                t2.interrupt();  //强制唤醒
    //            t.changeFlag(); //同时改变标志为位
            }
        }
    }
}

                }
            }
        };
//        t.join();
        t.start();

        Runnable r = new Runnable()
        {
            @Override
            public void run()
            {
                for (int i = 0; i < 50; i++)
                {
                    System.out.println("x=" + i);
                }
            }
        };
        new Thread(r).start();

        for (int i = 0; i < 50; i++)
        {
            System.out.println("x=" + i);
        }
    }
}
