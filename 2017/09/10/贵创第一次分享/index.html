重载与重写

概念

  • 重载(overload)：函数之间函数名相同但是参数的类型或者个数不同，与返回值无关
  • 重写(override)：也叫覆盖或者复写，是Java中override的不同翻译方式罢了；是指在继承或者实现关系中，子类方法与父类方法方法名，参数列表，返回值等完全相同，实现不同(其实这里的定义不是很准确，具体见后面)

来源

  • 两者都是Java中多态性的体现
  • 其实C语言中是没有重载和重写的概念的，因为编译器的原因，C语言并不支持函数同名；重载和重写随着面向对象的兴起和多态的需求，才逐渐开始占据比较重要的位置

    详解

    重载解析

  • 首先提一些最基本的：不能依据方法的返回值来区分重载；不能依据方法的访问权限来区分重载；成员变量之间没有重载的说法等

  • 重载主要是建立在同一个类的不同方法上，这里之所以说是主要，是因为在继承关系中，当子类方法与父类方法满足重载的关系时，这也是重载，而非覆盖。如下代码；其实这种现象的出现是继承时子类也会拥有父类的public属性和方法

-

class Father
{
    public void show()
    {
        System.out.println("Father Show");
    }
}
class Child extends Father
{
    public void show(int i)
    {
        System.out.println("Child Show");
    }
}
  • 如下函数的重载，为了遵循重载的准确性原则，所以即使null的值都可以赋值给Object和String,但是由于String继承于Object,所以最终调用的是带String参数的函数；所谓的准确性原则通俗一点来说就是，从下往上找，先找到的符合，或者在不具备继承关系之间的，就调用赋值最紧凑的那个

-

class Demo
{
    public void show(Object o)
    {
        System.out.println("Object");
    }
    public void show(String s)
    {
        System.out.println("String");
    }
    public void show(int i)
    {
        System.out.println("int");
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Demo demo = new Demo();
        demo.show(null); //输出String
    }
}
  • 但是对于下面这种调用时就会出现调用不确定的情况，这是因为编译器在String和String[]之间找不到更合适者

-

class Demo
{
    public void show(String s)
    {
        System.out.println("String");
    }
    public void show(String[]s)
    {
        System.out.println("String[]");
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Demo demo = new Demo();
        demo.show(null);  //报错
    }
}

  • 对于参数不定的函数重载，如下代码，其实编译器在处理不定参数的时候，是处理为一个数组；所以下面的输出中，依据重载的准确性原则，下面按照值的符合更紧凑来调用；若在下面再加一个重载

    public void show(String[] s){…}

这样的话会报错，和show(String...s)冲突，这也说明了编译器对于不定参数是处理为数组的

-

class Demo
{
    public void show(String s)
    {
        System.out.println("String");
    }
    public void show(String...s)
    {
        System.out.println("String...");
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Demo demo = new Demo();
        demo.show("h");            //String
        demo.show();            //String...
        demo.show(new String[]{"hell","he"});//String...
//        demo.show(null);        //报错
    }
}
  • 关于静态方法之间和静态方法与非静态方法之间的重载，可以视静态方法等效为非静态处理

重写解析

  • 重写上面的概念说要返回值类型，参数类型，函数名等完全一样，这中说法不太准确，参见下面代码

-

class Father
{
    public Object show()
    {
        System.out.println("Object show");
        return null;
    }
}
class Child extends Father
{
    public String show()
    {
        System.out.println("String show");
        return null;
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Child child = new Child();
        child.show();//String show
    }
}
  • 在Java核心编程中的解释是：允许子类将覆盖方法的返回类型定义为原返回类型的子类型；其实也可以这样理解，在Child中继承了父类返回值为Object的show方法，但是在子类还有一个返回值为String类型的show方法，因为不能依据函数的返回值类型来定义函数的重载，所以这里应该是重写(覆盖)；需要注意的是这种情况只有在子类方法返回值是父类待重载方法返回值的子类才行，不能父类待重载方法的返回值是子类返回值的子类，如下代码报错

-

class Father
{
    public String show()
    {
        System.out.println("Father show");
        return null;
    }
}

class Child extends Father
{
    public Object show()  //报错
    {
        System.out.println("Child show");
        return null;
    }
}
  • 子类方法要覆盖父类方法，必须保证权限大于等于父类权限，如果父类方法为private的话，那么子类根本就不能继承这个函数，也就谈不上覆盖了
  • 静态方法之间不存在覆盖的说法，覆盖准确的说是要针对非静态的成员函数之间；继承关系中的同名同参同返回值的静态方法之间只是一种隐藏，因为静态方法与对象无关，通过对象来调用静态方法其实意义不大验证方法是：在IDEA中我们都知道通过快捷键Control+o可以快速定位需要重写的方法，但是当父类中有一个静态方法时，如果想要在子类中去”重写”这个静态方法的话，通过Control+o的形式是找不到该静态方法的，但是对于非静态的方法却可以这样通过快捷键的方式来重写，这就验证了父类中的静态方法是不能override的

-

class Father
{
    public static void show()
    {
        System.out.println("Object show");
    }
}

class Child extends Father
{
    public void show()//报错
    {
        System.out.println("Child show");
    }
}
  • 还有同样的是成员变量之间不存在覆盖，覆盖是非静态成员函数之间的说法

多态中的调用问题

多态的出现使得程序的扩展性增强了，但是当需要先上转型或者向下转型的时候，各成员变量和成员函数的调用规律是怎样的呢？还有对于静态成员的调用是否有区别呢?

转型

  • 先上转型：父类引用指向子类对象(Father f = new Child();)
  • 向下转型：子类引用指向父类对象(Child c = new Father();)

多态中成员变量的调用

  • 下面的继承关系中，使用默认权限修饰符(子类和同包中可以访问)，子类继承了父类的成员变量m，此时对子类对象向上转型的时候，用父类引用调用该成员变量，输出的是父类中的值(虽然子类中有一个同名的成员变量)；可以验证的是静态成员变量调用规律相同
  • 总结起来就是：当先上转型的时候，调用成员变量，看等号的左边

-

class Father
{
    int m = 0;
}
class Child extends Father
{
    int m = 1;
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Father father = new Child();
        System.out.println(father.m);//输出0
    }
}

多态中成员函数的调用

  • 下面的向上转型调用成员函数的情况，调用的实际是子类中的成员方法(相当于覆盖了，因为成员变量之间不存在覆盖的说法)
  • 总结起来就是：向上转型时，调用成员函数，看等号的左边

-

class Father
{
    public void show()
    {
        System.out.println("Father show");
    }
}
class Child extends Father
{
    public void show()
    {
        System.out.println("Child show");
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Father father = new Child();
        father.show();//输出Child show
    }
}

多态中静态成员方法的调用

  • 虽然静态成员方法与对象无关，但是这里为了做一个讲解，仍用对象去调用静态成员方法，如下所示，实际调用的是父类中的静态成员方法
  • 总结起来就是：多态调用时，静态方法的调用看等号的左边

-

class Father
{
    public static void show()
    {
        System.out.println("Father show");
    }
}
class Child extends Father
{
    public static void show()
    {
        System.out.println("Child show");
    }
}
public class OverloadDemo
{
    public static void main(String[] args)
    {
        Father father = new Child();
        father.show(); //输出Father show
    }
}

多态调用的解释

  • 首先需要区别多态的两种形式：编译时多态和运行时多态；编译时多态就是上面所说的重载，是指在编译过程中，通过函数的参数类型，参数个数等来确定调用的是哪个方法；运行时多态又叫动态绑定，就是上面所说的覆盖，是指在运行时由JVM确定具体引用的是哪个类的对象，然后确定方法的调用，运行时多态依赖于具体的对象
  • 因为成员变量之间不存在所谓的多态，也不存在覆盖的说法，那么它就不需要根据运行时的具体对象去确定应该调用哪个类中的成员变量了，只需要根据引用类型即可确定
  • 调用静态方法时，由于静态方法与对象无关，它是随着类的加载而加载，所以对于静态方法的调用实际上就不需要实例化，那么就不存在多态的说法(静态方法的”覆盖”本质上只是一种隐藏)；在转型中调用静态方法时，实际上在运行时Java虚拟机已经确定好了方法所属的类，就不会再去检查引用实际指向的类对象是哪个了
  • 对于非静态方法的先上转型调用，此时需要根据所引用对象的具体类型去确定调用的是哪个方法

最后

按值传递还是按引用传递？

  • 最后提一下的是在Java中参数传递时实际上是没有引用传递的，都是按值传递；平时所说的三大引用类型的传递，实际上也是按值传递，只不过引用变量中存储的是一个地址罢了，此时传递的实际上是一个地址值

数组长度为0与数组指向null

  • 在实际应用中，为了增强程序的健壮性，通常遇到数组的时候，为了避免出现空指针异常，通常会先判断是否为null；但是更为常见的判断是
if(num==null||num.length==0){...}
  • 首先需要注意的是上面的判断顺序最好不要改变，否则的话会出现空指针异常，如下

-

public class OverloadDemo
{
    public static void main(String[] args)
    {
       int[] num = null;
       if(num.length==0||num==null)//NullPointerException
           System.out.println();
    }
}
  • 出现上面这种情况的原因其实也很容易想到，稍微注意一下其实就知道，当num指向null的时候，按照执行顺序会先执行num.length==0的判断，就是这种情况下导致了NullPointerException
  • 对于数组为空和数组指向null的区别，参见下面代码；null可以用于引用类型的初始化，但是其与数组为空的区别是：数组虽然为空，但是其还是一个对象，在堆内存还是分配了一个首地址；而对于null，更多的是对JVM垃圾回收机制的一个提醒，意味着该对象可以被回收

-

int[] num = null;//数组指向null，不可操作
int[] num = new int[0];//数组长度为0，可以一定程度上操作
  • 关于的此的相似的情况还有字符串中""和null的区别
  • 所以当一个函数需要返回数组时，一般都是返回的空数组null，这是为了方便调用者；但是，身为调用着，调用时还是应该先判null
  • 其实这里更推荐的写法应该如下，也就是说将常量值置为左直；但是笔者认为，这个在Java中其实已经有了一定程度上的优化，因为对于类似的判断语句，即使写为if(num=null)，编译器是会报错的；但是对于类似C语言样的编译器，为了防止误写，还是应该将常量置为左值，因为它的编译器允许这样的操作

-

int[] num = null;
if(null==num||0==num.length)
