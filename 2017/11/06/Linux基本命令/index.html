前言

Linux的基本操作和一些Linux常识笔记

一、Linux命令的基本格式

  1. [root@localhost ~]#

其中root表示的是当前为超级用户登陆
localhost表示的是主机名
~表示的是当前所在的目录（家目录，超级用户家目录在根目录下的一级目录，普通用户的家在home下的二级目录）

#表示超级用户输入提示符；$为普通用户输入提示符

  1. 命令格式：

命令 [选项] [参数]
注：有个别命令不遵循这种格式，但是极少
选项：

  1. ls [选项] [文件或目录]

查询目录内容
选项：（选项命令可以组合，如：-lh）

  • -a 显示所有文件，包括隐藏文件（.开头的文件是隐藏文件）
  • -l 显示详细信息(注意ls -l也可以简写为ll)
    • -d 查看目录属性
    • -i 显示inode（每个文件都有一个ID号）
      *　-h 人性化显示文件的大小

-rw-r–r–.

  • - Linux中以第一位区分文件类型，-表示文件，d表示目录，|表示软链接文件（快捷方式）
    后面的每三位一组

  • rw- 表示所有者（u）

  • r– 表示所属组（g）
  • r– 表其他人（o）

    注：

    r 读
    w 写
    x 执行
    也就是说Linux中用户对文件就这几种操作

二、文件处理命令

  1. 目录处理命令：

（Linux中的目录相当于Windows中的文件夹）

  • mkdir -p [目录名]
    建立目录，-p表示递归创建，也就是说要一次性创建多级目录时要加-p表示递归创建，也就是说要一次性创建多级目录时要加-p
    如: mkdir -p Hust/yuan
    若只是创建一级目录，则可以不加-p

  • cd [目录名]
    切换所在目录

  • cd ~ 进入当前用户的家目录（或者cd直接回车）
  • cd - 进入上次目录
  • cd .. 进入上一级目录

  • cd . 进入当前目录

    注：

    可以用tab键补全目录或者命令，可以多按两下，会自动列出

  • pwd
    显示当前所在路径

  • rmdir [目录名]
    删除空目录,注意这个只能删除空目录

  • rm -rf [目录名或文件名]
    这样可以强制删除目录或文件，但是这样删除要小心，因为他没有系统提示

  • rm -r会有提示

    注意：

    rm -rf / 这个表示的是删除根目录，很危险咯！
    所以一般不要用-rf

  • cp [选项] [原文件或目录] [目标目录]
    复制文件
    选项：

  • -r 复制目录
  • -p 连带文件属性复制
  • -d 若原文件是链接文件，则复制链接属性

  • -a 相当于-pdr

  • mv [原文件或目录] [目标目录]
    剪切或改名（如果原文件和目标文件在同一个目录下就是改名，不在同一目录下为剪切）

注：

可以在家目录root或者home（超级用户是root）,以及temp目录下随便放内容
一般只把比要数据放在根目录下
  1. 链接处理命令
  • ln -s [原文件] [目标文件]
    生成链接文件，-s表示生成软链接
    硬链接：
    • 硬链接相当于一个文件的两个不同接入点（可以通过不同的方式访问查询）
    • 不能跨分区
    • 不能针对目录使用
    • 只能通过i节点识别
      软链接：
    • 类似Windows创建快捷方式
    • 软链接拥有自己的I节点和Block块，但是数据块中只保存文件的文件名和I节点号
      无实际数据
    • 修改任意文件，另一个都改变
    • 删除原文件，软链接不能使用
    • 软链接的权限都是lrwxrwxrwx （第一个l表示软链接）,但是实际上由于二者的关系，
      软链接的访问权限还受到原文件的限制

三、文件搜索命令

  1. locate 文件名

    • 文件搜索速度很快

    • locate是在后台数据库中按文件名搜索，即在/var/lib/mlocate中搜索，所以对于新建的文件
      是不能用locate来搜索到的，因为locate的后台数据库更新有一定的时间间隔，但是可以用命令

    • updatedb来更新一下就可以搜索到了
      还可以用locate locate搜索自身
      但是locate也有缺点：只能以文件名来搜索
      配置文件：/etc/updatedb.conf 这个里面是设置locate忽略搜索哪些地方
    • PRUNE_BIND_MOUNTS = “yes” 开启搜索限制
    • PRUNEFS = 搜索时不搜索的文件系统
    • PRUNENAMES= 搜索时不搜索的文件类型
    • PRUNEPATHS = 搜索时不搜索的路径

  2. whereis 命令

    • 命令的搜索命令，搜索命令的位置，帮助文档的位置
      （其实可以类比，（whoami）就表示登陆者是谁；（whatis 命令）就会显示命令是干什么的）
    • whereis -b 命令 只查找可执行文件位置
    • whereis -m 命令 只查找帮助文件的位置
    • whereis只能查系统命令

  3. which 命令

    • 显示出命令的别名（如果有）和命令的位置
      PATH环境变量：定义的是系统搜索命令的路径

  4. find [搜索范围] [搜索条件]

    • 功能最强大，避免大范围搜索，会非常耗费系统资源
      find搜索是完全匹配搜索，若果要不完全搜索，就要用通配符
      • 表示匹配任意内容
    • ？ 表示匹配任意一个字符
    • [] 表示匹配任意一个中括号内的字符

| 如：find /root -name”ab[cd]” 注意这里要加””
find /root -name”*[cd]” 表示列出所有已c或者d结尾的命令

find /root -iname XXX 表示忽略大小写搜索
find /root -nouser 查找所有没有所有者的文件

find /var/log/ -mtime +10

  • -10 表示10天内修改文件
  • 10 表示10天当天修改的文件
  • +10 表示10天前修改的文件
  • atime 文件的访问时间
  • ctime 改变文件的属性
  • mtime 修改文件的内容

find . -size 25k（k小写）

  • 按文件的大小查找文件（这里查找的是等于25k的文件）(也可以按照兆(M)(注意大写))
    也可以+25k,-25k，意义明显哈

find . -iname 22342

  • 查找i节点是22342的文件（每个文件都有一个i节点）

find /etc -size +20k -a -size -50k

  • -a 表示的是and，两个条件都要满足
  • -o 表示or，二者之一

    注意这里的+20k，-50k表示的是大于20k，小于50k

find /etc -size +20k -a -size -50k -exec ls -lh {} \;

  • 查找etc目录下大于20k小于50k的文件并显示详细信息
    ##（注意{} \;是标准格式，只要写了exec就要写这个）

四、字符串搜索命令

  1. grep [选项] 字符串 文件名
    • 注意字符串要用””括起来，表示在文件中查找匹配符合条件的字符串
  • 选项：
    • -i 忽略大小写
    • -v 排除指定字符串
    • （find的是完全匹配，grep是不完全匹配，find是搜索文件名，grep是搜索文件内容）

五、帮助命令

  1. man 命令
    • 显示命令的帮助信息
    • man的查看还要分级别，可以用
    • man -f 命令 来显示改命令有哪些帮助文件等级
      　* man -1 命令 可以查看1等级的帮助

man -k 命令

  • 相当于 apropos 命令 表示查看和命令相关的所有帮助

六、其他帮助命令

  1. 命令 –help

    • 获取命令选项的帮助（只是显示有哪些选项 ）

  2. help shell内部命令

    • 获取shell内部命令。

  3. info 命令

      • 回车 进入子帮助页面（带有*号标记）
      • n 进入下一个帮助小节
      • p 进入上一个帮助小节
      • q 退出

七、关机和重启命令

  1. shutdown [选项]时间

    • 选项有：

      • -c 取消前一个关机命令
      • -h 关机
      • -r 重启
        | 当时间为now的时候表示立刻执行
        可以在最后加一个&表示后台执行，否则会卡在终端
        当卡在终端的时候可以用Ctrl+C终止

  2. reboot

    • 也可以重启

  3. runlevel

    • 查询当前系统的运行级别

  4. cat /etc/inittab

    • cat表示查看内容

  5. logout

    • 退出登录命令
